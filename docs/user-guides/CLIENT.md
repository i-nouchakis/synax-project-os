# Client User Guide

Complete guide for client users accessing Synax.

---

## Overview

As a **Client** user, you have read-only access to monitor your project's progress. You can:

- View project status and progress
- Monitor installation completion
- View client reports
- Create and track issues
- See floor plans and assets

---

## Quick Reference

### What You Can Do

| Category | Actions |
|----------|---------|
| Dashboard | View assigned projects |
| Projects | View progress |
| Floors | View floor plans |
| Rooms | View status |
| Assets | View installation status |
| Issues | Create, view, comment |
| Reports | View client reports |

### What You Cannot Do

| Action | Reason |
|--------|--------|
| Create projects | PM/Admin only |
| Create/edit floors | PM/Admin only |
| Edit rooms | Team only |
| Edit assets | Team only |
| Complete checklists | Team only |
| Manage inventory | Team only |
| View internal reports | Team only |

## Getting Started

### Logging In

1. Open Synax in your browser
2. Enter your email and password
3. Click **Login**

### Dashboard Overview

Your Dashboard shows:
- **Your projects** - Projects you have access to
- **Progress statistics** - Overall completion
- **Recent activity** - What's happening

---

## Viewing Your Project

### Project Overview

1. Click on your project from Dashboard
2. You'll see:
   - Project details (dates, status)
   - Floor list with room counts
   - Team members
   - Issue summary

### Understanding Project Status

| Status | Meaning |
|--------|---------|
| PLANNING | Project setup in progress |
| IN_PROGRESS | Installation work ongoing |
| ON_HOLD | Temporarily paused |
| COMPLETED | All work finished |

### Progress Metrics

Look for these indicators:
- **Room completion %** - How many rooms done
- **Asset status** - Equipment installed
- **Checklist completion** - Quality checks done
- **Open issues** - Problems being resolved

---

## Viewing Floors

### Floor List

From your project page:
1. Scroll to **Floors** section
2. See list of all floors
3. Click floor name to view details

### Floor Details

On the floor page you can see:
- Floor name and level
- Room list with status
- Floor plan (if available)
- Progress statistics

### Using Floor Plans

If a floor plan is uploaded:
1. View the interactive canvas
2. Use mouse scroll to zoom
3. Click and drag to pan
4. Click pins to see room details

### Understanding Pin Colors

| Color | Room Status |
|-------|-------------|
| Gray | Not started yet |
| Blue | Work in progress |
| Green | Completed |
| Red | Blocked/problem |

## Reporting Issues

### When to Report

Create an issue when you notice:
- Something not working
- Damage or defects
- Missing items
- Safety concerns
- Questions about work

### Creating an Issue

1. Go to **Issues** page
2. Click **New Issue**
3. Fill in:

| Field | Description |
|-------|-------------|
| Title | Brief summary |
| Description | Detailed explanation |
| Priority | How urgent is it |
| Project | Your project |
| Room | Where is the problem (optional) |

4. Click **Create**

### Priority Guidelines

| Priority | Use When |
|----------|----------|
| LOW | Minor observation |
| MEDIUM | Should be addressed |
| HIGH | Needs quick attention |
| CRITICAL | Urgent problem |

### Tracking Issues

1. Go to **Issues** page
2. See all issues for your projects
3. Click an issue for details
4. View status and comments

### Adding Comments

You can participate in issue discussions:
1. Open issue detail
2. Scroll to comments
3. Type your comment
4. Click **Add**

## Viewing Reports

### Available Reports

As a client, you can view:
- **Summary Report** - High-level overview
- **Client Report** - Detailed progress report

### Generating a Report

1. Go to **Reports** page
2. Select your project
3. Click **Client** tab
4. Review the report on screen

### Report Contents

The Client Report includes:
- Project summary
- Overall progress percentage
- Floor-by-floor breakdown
- Asset installation status
- Issue summary
- Key milestones

### Downloading Reports

1. Click **Export PDF**
2. Wait for generation
3. Click **Download**
4. Save to your computer

### Report History

1. Click **History** button
2. View previously generated reports
3. Download any past report

## Communication Best Practices

### Asking Questions

1. Create an issue for questions
2. Use MEDIUM priority
3. Be specific about what you need
4. Include room/asset reference if applicable

### Reporting Problems

1. Create issue with appropriate priority
2. Describe the problem clearly
3. Include location details
4. Add photos if possible (via mobile)

### Providing Feedback

1. Use issue comments for feedback
2. Be specific and constructive
3. Reference specific rooms/assets
4. Ask for clarification if needed

## Understanding Progress

### What "Completed" Means

A room is COMPLETED when:
- All assets are VERIFIED
- All checklists are 100%
- No blocking issues
- Documentation done

### Progress Calculation

Overall progress considers:
- Room completion status
- Asset verification
- Checklist completion
- Issue resolution

### What Affects Progress

Progress slows when:
- Issues block work
- Materials unavailable
- Access restricted
- Technical problems

## Frequently Asked Questions

### Q: Why can't I edit anything?

A: Client accounts have read-only access to ensure data integrity. Create an issue for any changes needed.

### Q: How do I know if work is on schedule?

A: Check the progress percentages and compare to project dates. The PM will communicate any delays.

### Q: Can I see detailed technical information?

A: Client reports show summary information. For technical details, contact your Project Manager.

### Q: How do I request a meeting/walkthrough?

A: Create an issue with LOW priority titled "Meeting Request" and include your availability.

### Q: What if I find a problem during a site visit?

A: Create an issue immediately with appropriate priority. Include photos if possible.

### Q: How often is the data updated?

A: Data updates in real-time as technicians complete work. Refresh your browser for latest data.
